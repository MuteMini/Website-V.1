Object Oriented Programming

What is an Object?

In Java, an object is a instance of a class, which is a template for the object for it to set up variables and methods to attach to the object.

Objects have attributes (instance variables) and behaviours (methods). These objects can be compared to real world objects, hence the reason it is called "objects".

Inheritance

Objects can take attributes and behaviours of another class. This is called Inheritance.

Image from Geeks for Geeks

The class supplying the attributes and behaviours is called the superclass and the one recieving it is called the subclass.

This allows similar objects to share the exact same variables and methods.

In Java, you inherit a class by using the extends keyword.

public class Min extends Person //The class Min extends another class called Person.

Polymorphism

Polymorphism is the action of having the same take performed in different ways through inheritance. Here is an example:

public class Fruit{
    public void rot(){
        System.out.println("oh no, i am rotting.");
    }        
}

public class Banana extends Fruit{
    public void rot(){
        System.out.println("crap, now i am all mushy!");
    }        
}

public class Strawberry extends Fruit{
    public void rot(){
        System.out.println("aww, i dont want to die...");
    }        
}

Now, when each of these classes run the method rot, they all print out separate things. This is the basic of polymorphism, and it allows for interesting program management.

Abstraction

The user doesn't want to see every single gunk in your code. This is where abstraction comes into play.

Abstractions allow the programmer to hide away details, allowing you to focus on what the object does instead of it's implementation.

Abstractions can be achieved in two ways, through abstract classes or interface.

An abstract class allows the programmer to have a mix of abstraction, so that certain methods are hidden while others aren't.

public abstract class Country{ //makes an abstract class
    
    /* Makes an abstract method; no method body is created.
     * This means the class that extends this Country class must 
     * code it's own checkMoney method.
     */
    public abstract int checkMoney(); 
    
    /* This method is not abstract; it has a method body. 
     * It has not achieved full abstraction.
     */
    public void goToWar(){
        System.out.println("I AM GOING TO FIGHT!"); 
    }        
}

A class that is an interface allows for 100% abstraction in its code. There cannot be any method bodies.

public interface class Country{ //makes an abstract class
    
    /* Makes two method; no method body is created.
     * Full abstraction has been achieved.
     */
    public int checkMoney(); 

    public void goToWar();
}

Encapsulation

In encapsulation, everything about a class is hidden outside of itself and the methods that inherit it.

Encapsulation can be achieved by setting all of the instance variables as private. Then, to access and change the variables of that class, you must make accessor and mutator methods.
